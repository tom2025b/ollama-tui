# ai-suite-gui (egui Desktop GUI) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a native egui desktop chat GUI as a new `ai-suite-gui` workspace crate, backed by a thin public streaming API added to the `ai-suite` library.

**Architecture:** A new `pub async fn stream_prompt(prompt, context, on_token) -> Result<(String, String)>` is added to the `ai-suite` library, wrapping the existing `pub(crate)` routing + provider pipeline. The GUI crate (`ai-suite-gui`) calls this from a Tokio task, forwards tokens over an `mpsc` channel to the egui render loop, and displays them as chat bubbles. The existing `ai-suite-cli` binary is untouched.

**Tech Stack:** Rust 2024 edition, `eframe`/`egui` 0.28, `tokio` 1.x (multi-thread runtime managed manually alongside eframe), `tokio::sync::mpsc` for cross-thread events.

## Task 2: Add `stream_prompt` public API to the library

**Files:**
- Create: `ai-suite/src/stream.rs`
- Modify: `ai-suite/src/lib.rs`

### Background

`providers::execution::stream_model_request` and `routing::ModelRouter` are `pub(crate)`. Because `stream.rs` lives inside the `ai-suite` crate, it can call them freely. We just need to expose the result publicly.

`stream_prompt` must have `+ Send` on the `on_token` bound so that the async future it returns is `Send` — required by `tokio::spawn` in the GUI crate.

- [ ] **Step 1: Create `ai-suite/src/stream.rs`**

```rust
use ai_suite::Result;

use crate::llm::ConversationTurn;
use crate::providers::execution::{ModelRequest, stream_model_request};
use crate::routing::{ModelRouter, Router};

/// Public entry point for sending a prompt through the full routing + provider
/// pipeline without going through the TUI.
///
/// Returns (full_response_text, model_name) so callers can display which model
/// was chosen.
pub async fn stream_prompt(
    prompt: String,
    context: Vec<ConversationTurn>,
    on_token: impl FnMut(String) + Send,
) -> Result<(String, String)> {
    let runtime = crate::runtime::Runtime::load();
    let router = ModelRouter::new(runtime.config().models());
    let decision = router.route(&prompt);
    let model_name = decision.model.name.clone();
    let request = ModelRequest::new(decision.model, context, prompt);
    let full_text = stream_model_request(&request, on_token).await?;
    Ok((full_text, model_name))
}
```

## Task 4: Implement `backend.rs`

**Files:**
- Create: `ai-suite-gui/src/backend.rs`
- Modify: `ai-suite-gui/src/main.rs` (add `mod backend;`)

### Background

`spawn_request` fires a tokio task that calls `ai_suite::stream_prompt` and forwards every token to the egui render loop via an `mpsc` channel. It also calls `ctx.request_repaint()` after each token so the render loop wakes up immediately rather than waiting for the next vsync or user event.

The `tokio::runtime::Handle` is passed in (not obtained from a global) because eframe owns the main thread — we cannot use `#[tokio::main]` and the only way to spawn tasks is through a handle to a runtime started before eframe.

- [ ] **Step 1: Create `ai-suite-gui/src/backend.rs`**

```rust
use ai_suite::{stream_prompt, ConversationTurn};
use egui::Context;
use tokio::{runtime::Handle, sync::mpsc};

/// Events streamed from the backend task to the GUI render loop.
pub enum BackendEvent {
    /// One streamed token from the model.
    Token(String),
    /// Response complete. Carries the full assembled text and the model name
    /// chosen by the router (e.g. "claude-sonnet-4-6").
    Done { full_text: String, model_name: String },
    /// An error occurred. The message is human-readable.
    Error(String),
}

/// Spawn a background tokio task that calls stream_prompt and forwards events
/// to the GUI via `tx`. Calls `ctx.request_repaint()` after each token so the
/// egui frame loop wakes up promptly.
pub fn spawn_request(
    prompt: String,
    context: Vec<ConversationTurn>,
    tx: mpsc::UnboundedSender<BackendEvent>,
    ctx: Context,
    handle: Handle,
) {
    handle.spawn(async move {
        let tx_tok = tx.clone();
        let ctx_tok = ctx.clone();

        // on_token is called by stream_prompt for each streamed token
        let result = stream_prompt(prompt, context, move |token| {
            let _ = tx_tok.send(BackendEvent::Token(token));
            ctx_tok.request_repaint();
        })
        .await;

        match result {
            Ok((full_text, model_name)) => {
                let _ = tx.send(BackendEvent::Done { full_text, model_name });
            }
            Err(e) => {
                let _ = tx.send(BackendEvent::Error(e.to_string()));
            }
        }
        ctx.request_repaint();
    });
}
```

## Task 5: Implement `app.rs`

**Files:**
- Create: `ai-suite-gui/src/app.rs`
- Modify: `ai-suite-gui/src/main.rs` (add `mod app;`)

### Background

`App` is the egui application state. `eframe::App::update` is called every frame — it drains the `mpsc` channel first (applying any new tokens to the last message), then renders the full UI. The render is a pure function of state; no persistent mutation happens outside `drain_channel` and `send_message`.

Color scheme is Catppuccin Mocha: background `#1e1e2e`, user bubbles `#45478a` (indigo), assistant bubbles `#313244` (surface1), error bubbles `#7a2525` (dark red), text `#cdd6f4`.

- [ ] **Step 1: Create `ai-suite-gui/src/app.rs`**

```rust
use ai_suite::ConversationTurn;
use egui::{Color32, Frame, Key, Layout, Margin, Modifiers, RichText, Rounding, ScrollArea};
use tokio::{runtime::Handle, sync::mpsc};

use crate::backend::{BackendEvent, spawn_request};

// ---------------------------------------------------------------------------
// Data types
// ---------------------------------------------------------------------------

#[derive(PartialEq)]
enum Role {
    User,
    Assistant,
}

struct ChatMessage {
    role: Role,
    content: String,
    /// False while the assistant is still streaming this message.
    complete: bool,
    /// True if the backend returned an error for this turn.
    is_error: bool,
}

// ---------------------------------------------------------------------------
// App state
// ---------------------------------------------------------------------------

pub struct App {
    messages: Vec<ChatMessage>,
    input: String,
    /// Model name shown in the top bar. Starts as "Ready", updated after each
    /// completed response.
    current_model: String,
    rx: Option<mpsc::UnboundedReceiver<BackendEvent>>,
    streaming: bool,
    handle: Handle,
}

impl App {
    pub fn new(handle: Handle) -> Self {
        Self {
            messages: Vec::new(),
            input: String::new(),
            current_model: "Ready".to_string(),
            rx: None,
            streaming: false,
            handle,
        }
    }

    // -----------------------------------------------------------------------
    // Logic helpers
    // -----------------------------------------------------------------------

    /// Build conversation context from all completed user/assistant pairs so
    /// far. This is passed to stream_prompt as bounded context.
    fn conversation_context(&self) -> Vec<ConversationTurn> {
        let complete: Vec<&ChatMessage> = self
            .messages
            .iter()
            .filter(|m| m.complete && !m.is_error)
            .collect();

        complete
            .chunks(2)
            .filter_map(|pair| {
                if pair.len() == 2
                    && pair[0].role == Role::User
                    && pair[1].role == Role::Assistant
                {
                    Some(ConversationTurn {
                        user: pair[0].content.clone(),
                        assistant: pair[1].content.clone(),
                    })
                } else {
                    None
                }
            })
            .collect()
    }

    /// Validate input, build context, push placeholder messages, and fire the
    /// backend request.
    fn send_message(&mut self, ctx: &egui::Context) {
        let prompt = self.input.trim().to_string();
        if prompt.is_empty() || self.streaming {
            return;
        }

        // Snapshot context BEFORE adding the new user message.
        let context = self.conversation_context();

        self.messages.push(ChatMessage {
            role: Role::User,
            content: prompt.clone(),
            complete: true,
            is_error: false,
        });

        // Empty placeholder — tokens will fill this in.
        self.messages.push(ChatMessage {
            role: Role::Assistant,
            content: String::new(),
            complete: false,
            is_error: false,
        });

        let (tx, rx) = mpsc::unbounded_channel();
        self.rx = Some(rx);
        self.streaming = true;
        self.input.clear();

        spawn_request(prompt, context, tx, ctx.clone(), self.handle.clone());
    }

    /// Drain all pending backend events from the channel into the message list.
    fn drain_channel(&mut self) {
        let Some(rx) = &mut self.rx else { return };

        while let Ok(event) = rx.try_recv() {
            match event {
                BackendEvent::Token(token) => {
                    if let Some(msg) = self.messages.last_mut() {
                        msg.content.push_str(&token);
                    }
                }
                BackendEvent::Done { full_text, model_name } => {
                    if let Some(msg) = self.messages.last_mut() {
                        msg.content = full_text;
                        msg.complete = true;
                    }
                    self.current_model = model_name;
                    self.streaming = false;
                    self.rx = None;
                }
                BackendEvent::Error(e) => {
                    if let Some(msg) = self.messages.last_mut() {
                        msg.content = format!("Error: {e}");
                        msg.complete = true;
                        msg.is_error = true;
                    }
                    self.streaming = false;
                    self.rx = None;
                }
            }
        }
    }

    // -----------------------------------------------------------------------
    // Rendering helpers
    // -----------------------------------------------------------------------

    fn render_top_bar(&self, ctx: &egui::Context) {
        let text_col = Color32::from_rgb(0xcd, 0xd6, 0xf4);
        let accent   = Color32::from_rgb(0x89, 0xb4, 0xfa);

        egui::TopBottomPanel::top("top_bar").show(ctx, |ui| {
            ui.horizontal(|ui| {
                ui.add_space(4.0);
                ui.label(RichText::new("ai-suite").strong().color(text_col).size(15.0));
                ui.with_layout(Layout::right_to_left(egui::Align::Center), |ui| {
                    ui.add_space(4.0);
                    ui.label(RichText::new(&self.current_model).color(accent).size(13.0));
                });
            });
        });
    }

    fn render_input_bar(&mut self, ctx: &egui::Context) {
        let text_col   = Color32::from_rgb(0xcd, 0xd6, 0xf4);
        let input_bg   = Color32::from_rgb(0x18, 0x18, 0x25);
        let btn_col    = Color32::from_rgb(0x89, 0xb4, 0xfa);
        let input_id   = egui::Id::new("main_input");

        egui::TopBottomPanel::bottom("input_panel").show(ctx, |ui| {
            ui.add_space(8.0);
            ui.horizontal(|ui| {
                // Consume Enter (without Shift) before TextEdit sees it,
                // so it sends the message rather than inserting a newline.
                let had_focus = ctx.memory(|m| m.has_focus(input_id));
                let enter_hit = had_focus
                    && ctx.input_mut(|i| i.consume_key(Modifiers::NONE, Key::Enter));

                let text_edit = egui::TextEdit::multiline(&mut self.input)
                    .id(input_id)
                    .hint_text("Type a message…  (Enter to send, Shift+Enter for newline)")
                    .desired_rows(1)
                    .desired_width(ui.available_width() - 52.0)
                    .text_color(text_col)
                    .frame(false);

                let te_frame = Frame::new()
                    .fill(input_bg)
                    .rounding(Rounding::same(6.0))
                    .inner_margin(Margin { left: 10.0, right: 10.0, top: 6.0, bottom: 6.0 });

                te_frame.show(ui, |ui| {
                    ui.add(text_edit);
                });

                let send_btn = ui.add_enabled(
                    !self.streaming,
                    egui::Button::new(RichText::new("▶").color(btn_col)).min_size([40.0, 32.0].into()),
                );

                if (enter_hit || send_btn.clicked()) && !self.streaming {
                    self.send_message(ctx);
                    ctx.memory_mut(|m| m.request_focus(input_id));
                }
            });
            ui.add_space(8.0);
        });
    }

    fn render_messages(&self, ctx: &egui::Context) {
        let bg          = Color32::from_rgb(0x1e, 0x1e, 0x2e);
        let text_col    = Color32::from_rgb(0xcd, 0xd6, 0xf4);
        let user_bg     = Color32::from_rgb(0x45, 0x47, 0x8a);
        let asst_bg     = Color32::from_rgb(0x31, 0x32, 0x44);
        let err_bg      = Color32::from_rgb(0x7a, 0x25, 0x25);

        egui::CentralPanel::default()
            .frame(Frame::new().fill(bg))
            .show(ctx, |ui| {
                ScrollArea::vertical()
                    .auto_shrink([false, false])
                    .stick_to_bottom(true)
                    .show(ui, |ui| {
                        ui.add_space(10.0);

                        for msg in &self.messages {
                            let bubble_bg = if msg.is_error {
                                err_bg
                            } else if msg.role == Role::User {
                                user_bg
                            } else {
                                asst_bg
                            };

                            // Append blinking cursor while streaming.
                            let text = if !msg.complete {
                                format!("{}▌", msg.content)
                            } else {
                                msg.content.clone()
                            };

                            let bubble_frame = Frame::new()
                                .fill(bubble_bg)
                                .rounding(Rounding::same(8.0))
                                .inner_margin(Margin { left: 12.0, right: 12.0, top: 8.0, bottom: 8.0 });

                            if msg.role == Role::User {
                                // Right-aligned user bubble
                                ui.with_layout(Layout::right_to_left(egui::Align::TOP), |ui| {
                                    bubble_frame.show(ui, |ui| {
                                        ui.set_max_width(ui.available_width() * 0.72);
                                        ui.label(RichText::new(&text).color(text_col).size(14.0));
                                    });
                                });
                            } else {
                                // Left-aligned assistant bubble
                                bubble_frame.show(ui, |ui| {
                                    ui.set_max_width(ui.available_width() * 0.88);
                                    ui.label(RichText::new(&text).color(text_col).size(14.0));
                                });
                            }

                            ui.add_space(6.0);
                        }

                        ui.add_space(4.0);
                    });
            });
    }
}

// ---------------------------------------------------------------------------
// eframe integration
// ---------------------------------------------------------------------------

impl eframe::App for App {
    fn update(&mut self, ctx: &egui::Context, _frame: &mut eframe::Frame) {
        // Apply events from the backend before rendering.
        self.drain_channel();

        // Keep repainting while streaming so the cursor blinks and tokens
        // appear promptly.
        if self.streaming {
            ctx.request_repaint();
        }

        // Dark theme base.
        ctx.set_visuals(egui::Visuals::dark());

        self.render_top_bar(ctx);
        self.render_input_bar(ctx);
        self.render_messages(ctx);
    }
}
```

## Task 6: Implement `main.rs`

**Files:**
- Modify: `ai-suite-gui/src/main.rs` (replace placeholder body)

### Background

eframe calls `run_native`, which takes over the calling thread for the event loop. Tokio's `#[tokio::main]` cannot be used because that macro also wants the main thread. The solution: build the tokio multi-thread runtime manually before calling `run_native`, then hand a `Handle` into the `App`. The runtime stays alive because it's bound to `_rt` in `main` scope.

- [ ] **Step 1: Replace `ai-suite-gui/src/main.rs`**

```rust
mod app;
mod backend;

use app::App;

fn main() {
    // Build the tokio runtime before eframe takes the main thread.
    // The `_rt` binding keeps the runtime alive until main() returns.
    let rt = tokio::runtime::Builder::new_multi_thread()
        .enable_all()
        .build()
        .expect("Failed to build tokio runtime");
    let handle = rt.handle().clone();
    let _rt = rt;

    let native_options = eframe::NativeOptions {
        viewport: egui::ViewportBuilder::default()
            .with_inner_size([860.0, 620.0])
            .with_title("ai-suite"),
        ..Default::default()
    };

    eframe::run_native(
        "ai-suite",
        native_options,
        Box::new(move |_cc| Ok(Box::new(App::new(handle)) as Box<dyn eframe::App>)),
    )
    .expect("Failed to run GUI");
}
```
